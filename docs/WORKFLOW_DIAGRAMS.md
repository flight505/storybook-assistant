## 7. AskUserQuestion Implementation Details

### Code Structure

```typescript
// In command files (e.g., setup-storybook.md)

AskUserQuestion({
  questions: [
    {
      question: "Which testing features would you like to enable?",
      header: "Testing",  // 12 char max chip/tag
      multiSelect: true,  // Allow multiple selections
      options: [
        {
          label: "Interaction Tests (Recommended)",
          description: "Test user interactions with Vitest + Playwright"
        },
        {
          label: "Accessibility Tests",
          description: "WCAG compliance checking with axe-core"
        },
        {
          label: "Code Coverage",
          description: "Track test coverage metrics with V8"
        }
      ]
    },
    {
      question: "Enable AI-powered visual asset generation?",
      header: "Visual Gen",
      multiSelect: false,  // Single selection (radio buttons)
      options: [
        {
          label: "Yes (Recommended)",
          description: "Generate style guides and component mockups"
        },
        {
          label: "No",
          description: "Skip visual generation features"
        }
      ]
    }
  ]
});

// User selections are returned as:
{
  "Testing": ["Interaction Tests (Recommended)", "Accessibility Tests"],
  "Visual Gen": "Yes (Recommended)"
}
```

### UI Characteristics

**Header (Chip/Tag):**
- Max 12 characters
- Acts as visual label
- Examples: "Testing", "Framework", "Build Tool"

**Question:**
- Full descriptive text
- Ends with question mark
- Clear and specific

**Options:**
- 2-4 options (not including automatic "Other")
- Label: Concise (1-5 words)
- Description: Explains choice and implications
- "Other" option automatically added for custom input

**Multi-Select:**
- `multiSelect: true` â†’ Checkboxes (â˜‘â˜)
- `multiSelect: false` â†’ Radio buttons (â—â—‹)

## Summary

### Key Patterns

1. **Auto-Detection â†’ User Confirmation â†’ Execution**
   - Detect what we can automatically
   - Ask user for preferences/decisions
   - Execute with progress tracking

2. **AskUserQuestion for Choices**
   - Used whenever there are multiple valid options
   - Provides context and recommendations
   - Allows custom input via "Other"

3. **TodoWrite for Progress**
   - Shows real-time task completion
   - Keeps user informed during long operations
   - Clear indication of what's happening

4. **Natural Language Support**
   - Skills auto-trigger on natural queries
   - No need to memorize commands
   - Agents work autonomously

### When AskUserQuestion is Used

- **Setup commands**: Framework choice, testing preferences, build tools
- **Generation commands**: Component selection, variant detection, style choices
- **Sync commands**: Conflict resolution, direction preference
- **Analysis commands**: Which components to analyze, deprecation decisions
